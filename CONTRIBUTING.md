# Contributing to SIBAL

First off, thank you for considering contributing to SIBAL! It's people like you that make SIBAL such a chaotic and fun tool for everyone.

## <img src="https://cdn.jsdelivr.net/gh/twitter/twemoji@14.0.2/assets/svg/1f4da.svg" width="20" height="20"> How Can I Contribute?

### Reporting Bugs
If you find a bug that makes the package *less* chaotic or fails to play the siren correctly, please open an issue.
- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see and what you saw instead.

### Suggesting Enhancements
Want more madness? More colors? More sounds? Suggest it!
- Explain why this enhancement would be useful.
- Provide specific examples of how it should work.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`python test_emergency.py`).
4. Make sure your code follows the existing style.
5. Issue that pull request!
