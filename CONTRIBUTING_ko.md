# SIBAL 기여 가이드라인

SIBAL 프로젝트에 기여해주셔서 감사합니다! 여러분의 기여가 이 패키지를 더욱 혼란스럽고 재미있게 만듭니다.

## <img src="https://cdn.jsdelivr.net/gh/twitter/twemoji@14.0.2/assets/svg/1f4da.svg" width="20" height="20"> 기여 방법

### 버그 보고
패키지가 덜 혼란스럽거나, 사이렌이 제대로 울리지 않는 버그를 발견하시면 이슈를 남겨주세요.
- 명확하고 설명적인 제목을 사용해주세요.
- 문제를 재현할 수 있는 정확한 단계를 설명해주세요.
- 기대했던 동작과 실제로 발생한 동작을 비교하여 설명해주세요.

### 기능 제안
더 큰 광기, 더 많은 색상, 더 시끄러운 소리를 원하시나요? 제안해 주세요!
- 왜 이 기능이 유용한지 설명해주세요.
- 어떻게 작동해야 하는지에 대한 구체적인 예시를 제공해주세요.

### 풀 리퀘스트 (Pull Requests)
1. 리포지토리를 포크하고 `main` 브랜치에서 새로운 브랜치를 만드세요.
2. 테스트가 필요한 코드를 추가했다면 테스트 코드를 포함해 주세요.
3. 테스트가 통과하는지 확인하세요 (`python test_emergency.py`).
4. 기존 코드 스타일을 준수해 주세요.
5. PR을 제출해 주세요!
